# 뱀서류 웹 게임 제작 기획안

> 작성일: 2026-07-14
> 대상: Phaser 3 + Vite 기반, 브라우저 배포 (Poki / CrazyGames / 자체 AdSense)

---

## 0. 뱀서류란 무엇인가 — 분해해서 보기

"뱀서류(Vampire Survivors-like)"는 겉보기엔 단순한 슈팅 같지만, 실제로는 **세 개의 루프가 겹쳐 돌아가는 구조**다. 이걸 먼저 이해해야 무엇을 만들어야 하는지 보인다.

### 루프 1 — 초 단위 (조작 루프)

플레이어가 하는 유일한 조작은 **이동**이다. 공격은 자동이다. 그래서 플레이어의 머릿속 질문은 "어디를 쏠까"가 아니라 **"어디로 도망가야 안 죽고 잘 맞출까"** 가 된다. 이게 뱀서류의 정체성이다. 조준을 넣는 순간 뱀서류가 아니라 트윈스틱 슈터가 된다.

### 루프 2 — 분 단위 (성장 루프)

적을 죽이면 **경험치 젬**이 떨어지고, 주우면 경험치가 차고, 레벨업하면 **카드 3장 중 1장을 고른다.** 새 무기를 얻거나 가진 무기를 강화한다.

**여기가 재미의 90%다.** 이 게임의 중독성은 총 쏘는 맛이 아니라 "이번 판엔 어떤 조합이 나올까"에서 나온다. 이게 없으면 아무리 잘 만들어도 그냥 지루한 무한 슈터다. 뒤에서 다시 강조한다.

### 루프 3 — 판 단위 (메타 루프)

죽으면 골드가 남고, 골드로 영구 강화를 산다. 다음 판은 조금 더 세게 시작한다. **"한 판만 더"를 만드는 장치**다. 다만 이건 후순위다 — 루프 2가 재미없으면 루프 3을 붙여도 소용없다.

### 그 외 필수 요소

- **시간 = 난이도**: 시간이 갈수록 적이 많아지고, 새로운 종류가 나오고, 보스가 등장한다.
- **압도적인 물량**: 화면에 적이 수백 마리. 이게 시각적 쾌감의 원천이자, **최대의 기술적 난관**이다.
- **짧은 세션**: 한 판 5~15분. 웹 게임은 더 짧아야 한다 (뒤에서 설명).

## 1. 현재 위치 — 이미 만든 것

프로토타입 (`src/main.js`) 에 다음이 들어가 있다:

- 이동 (WASD + 가상 조이스틱)
- 자동 조준 · 자동 발사 (가장 가까운 적)
- 적 체력, 넉백, 피격 표시
- 1분마다 스폰 1.5배
- 체력 / 무적 시간 / 게임오버 / 재시작

즉 **루프 1의 뼈대만 완성**됐다. 루프 2(성장)는 통째로 없다. 지금 상태로는 "재미있는지" 판단할 수 없다 — 판단의 근거가 될 부분이 아직 안 만들어졌기 때문이다.

## 2. 단계별 제작 순서

각 단계 끝에 **"여기서 멈춰도 게임이 돌아가는가?"** 를 만족하도록 쪼갰다. 이게 초보자가 미완성으로 끝나지 않는 유일한 방법이다.

### Phase 1 — 코어 루프 완성 ⭐ 가장 중요

> 목표: "한 판 더 하고 싶은가?"에 답할 수 있는 상태

| 할 일 | 왜 |
|---|---|
| 적 죽으면 **경험치 젬** 드랍 | 성장 루프의 입구 |
| 젬 자석 (일정 거리 안이면 빨려옴) | 없으면 줍는 게 노동이 된다 |
| 경험치 바 + 레벨업 | 진행감 |
| 레벨업 시 **게임 일시정지 → 카드 3장 중 1택** | 여기가 진짜 게임이다 |
| 업그레이드 5~6종 (데미지+, 쿨다운-, 관통+, 이동속도+, 최대체력+, 젬 획득범위+) | 조합의 최소 단위 |

**이 단계가 끝나면 반드시 멈추고 테스트한다.** 이유는 Phase 1 검증 항목에서 설명한다.

> ⚠️ 흔한 실수: 무기 종류부터 늘린다. 무기가 10개라도 성장 루프가 없으면 재미없고, 성장 루프가 있으면 무기 1개로도 재미있다. **순서를 지켜라.**

---

### Phase 2 — 무기와 적의 다양화

> 목표: 판마다 다른 경험 만들기

- **무기 3~5종** — 서로 "다르게 노는" 것이어야 한다. 데미지 숫자만 다르면 의미 없다.
  - 활 (직선, 관통) — 이미 있음
  - 근접 회전 (플레이어 주위를 도는 궤도체) — 붙는 적 대응
  - 장판/오라 (주변 지속 데미지) — 몰릴 때 강함
  - 유도 미사일 (느리지만 반드시 맞음)
- **적 3종** — 기본(느리고 약함) / 돌진(빠르고 약함) / 탱커(느리고 단단함)
- 무기 레벨 (같은 무기를 반복 강화, 최대 5레벨 등)

> ⚠️ 스코프 함정: 무기 20종을 상상하게 된다. **5종에서 끊어라.** 뱀서류의 재미는 무기 개수가 아니라 무기 *조합*에서 나오고, 5종이면 조합은 이미 충분히 많다.

---

### Phase 3 — 세션 구조

> 목표: "끝"이 있는 게임 만들기

- **러닝타임 확정** — 웹 게임은 **5~8분** 권장 (일반 뱀서는 30분. 너무 길다)
- 1분마다 새 적 등장, 특정 시점 **엘리트/보스**
- **클리어 화면** (버텼다!) + 사망 화면 (기록)
- 최고 기록 저장 (`localStorage`)

> ⚠️ **난이도 곡선의 함정 — 반드시 읽을 것**
> 현재 "1분마다 스폰 1.5배"는 복리다. 계산해보면:
> - 3분: 3.4배 · 5분: 7.6배 · 10분: 57배
>
> 10분이면 스폰 간격이 0.02초다. 게임이 아니라 재난이다. **상한을 걸거나(예: 최대 8배), 증가율을 낮춰야(1.3배) 한다.** 지금 값은 "5분 안에 죽는 게임" 기준이라는 걸 알고 써야 한다.

---

### Phase 4 — 게임 필 (Game Feel)

> 목표: 같은 게임을 "잘 만든 것처럼" 느끼게 하기

이 단계는 기능을 하나도 안 늘리지만 **체감 품질을 가장 크게 올린다.** 포털 심사 통과 여부가 여기서 갈린다.

- 화면 흔들림 (피격 시 짧게)
- 데미지 숫자 팝업
- 적 사망 이펙트 (터짐, 파티클)
- 레벨업 연출 (번쩍)
- **사운드** — 발사 / 명중 / 레벨업 / 피격. 무료 SFX면 충분 (freesound, kenney.nl)
- 화면 정지(hitstop) — 타격 순간 2~3 프레임 멈춤

> 도형만 써도 게임 필이 좋으면 "미니멀한 디자인"으로 읽히고, 게임 필이 없으면 "성의 없음"으로 읽힌다. **같은 그래픽인데 평가가 갈리는 지점이다.**

---

### Phase 5 — 성능 최적화 ⚠️ 뱀서류의 진짜 난관

> 목표: 적 300마리에서 60fps

뱀서류는 **화면에 적이 수백 마리 있는 게 정체성**이다. 그런데 Phaser에서 `this.add.rectangle()`을 300번 부르고 매 프레임 300개를 순회하면 모바일에서 프레임이 무너진다.

필요한 기법:

- **오브젝트 풀링** — 적/화살/젬을 `destroy()` 하지 말고 재사용. 생성·파괴 비용이 가장 크다.
- **충돌 검사 최적화** — 지금은 화살마다 모든 적을 검사한다(O(n×m)). 적 300 × 화살 20 = 6000회/프레임. 공간 분할(그리드)로 줄인다.
- **경험치 젬 병합** — 젬 100개 굴러다니면 그것만으로 렉. 일정 개수 넘으면 합친다.
- 화면 밖 적은 업데이트 스킵

> ⚠️ **단, 조기 최적화 금지.** Phase 1~4는 적 50마리 수준이라 최적화 없이도 잘 돈다. **실제로 느려진 걸 눈으로 본 다음에** 손대라. 미리 풀링을 넣으면 코드만 복잡해지고 게임은 안 만들어진다.

---

### Phase 6 — 배포와 수익화

- 파일 크기 **20MB 이하**, 로딩 **3초 이내** (Phaser만 320KB gzip — 이미지 안 쓰면 여유롭다)
- 모바일 세로/가로 대응 확인
- **CrazyGames** 먼저 (진입 장벽 낮음) → 반응 좋으면 **Poki** (심사 까다롭지만 수익 큼)
- 포털 SDK 통합 (광고 삽입 지점: 게임오버 후, 부활 시)
- 또는 자체 사이트 + AdSense

## 3. 주의할 점 (실패 패턴)

### 제작 순서 관련

- **성장 루프를 뒤로 미루지 마라.** 이게 게임의 심장이다. 무기·적·그래픽보다 먼저다.
- **한 번에 하나씩 완성하라.** "무기 만들다 말고 적 만들다 말고 UI 만들다 말고" = 영원히 미완성.
- **매 Phase마다 실행되는 상태를 유지하라.** 깨진 채로 이틀 이상 두지 마라.

### 게임 디자인 관련

- **조준을 넣고 싶은 유혹** — 넣는 순간 장르가 바뀐다. 자동 공격을 유지하라.
- **난이도를 본인 기준으로 맞추지 마라.** 만든 사람은 이미 고수다. 처음 하는 사람은 1분도 못 버틴다.
- **첫 30초가 전부다.** 웹 게임은 재미없으면 30초 만에 닫는다. 시작하자마자 적이 오고, 바로 쏘고, 30초 안에 첫 레벨업이 와야 한다.

### 기술 관련

- **최적화는 느려진 다음에.** (위 참조)
- **에셋 욕심 금지.** 도형으로 시작해서 게임이 재미있어지면 그때 그래픽을 얹는다. 반대 순서로 가면 예쁜 쓰레기가 나온다.
- **저작권** — 게임 *메커니즘*은 저작권이 없다. 뱀서류를 만드는 건 완전히 합법이다. 다만 **이름, 캐릭터, 음악, 스프라이트를 베끼면 안 된다.** 에셋은 CC0(kenney.nl 등)만.

### 수익화 관련

- **첫 게임으로 돈 벌 생각을 하지 마라.** 첫 게임의 목표는 "완성"이다.
- 포털 수익은 **플레이 시간 × 재플레이율**에 비례한다. 화려함보다 "한 판 더"가 돈이 된다.

## 4. 검증 — 만들기 전에 확인하기

> 프로젝트 허브의 원칙: **"만들기 전에 결제 의사를 확인한다"**

게임은 광고 수익 모델이라 "결제 의사"를 직접 물을 수 없다. 대신 같은 원리를 **재플레이 행동**으로 치환한다. 관심 표명("재밌겠네요")이 아니라, **말없이 재시작 버튼을 누르는 행동**이 진짜 신호다.

### Phase 1 직후 — 5명 룰 (게임 버전)

Phase 1이 끝나면 **더 만들지 말고** 5명에게 링크를 보낸다. Vercel에 이미 자동 배포되니 링크 하나면 된다.

**아무 설명도 하지 말고, 옆에서 보기만 한다.**

| 관찰 항목 | 통과 기준 |
|---|---|
| 30초 안에 조작을 이해하는가 | 5명 중 4명 |
| 죽은 뒤 **말 안 해도** 재시작을 누르는가 | 5명 중 3명 |
| 자발적으로 3판 이상 하는가 | 5명 중 2명 |

- ❌ 나쁜 질문: "재밌어?" → 다들 재밌다고 한다 (악의 없는 거짓말)
- ✅ 좋은 관찰: 재시작 버튼을 **누르는지 안 누르는지** 그냥 본다

### 실패 기준선 (미리 못 박아두기)

> **5명 중 3명이 자발적으로 재시작하지 않으면, Phase 2로 넘어가지 않는다.**

이 경우 선택지는 두 가지다: (1) 코어 루프를 고친다 (업그레이드 선택이 심심한 게 원인일 확률이 높다), (2) 접는다. **무기를 늘려서 해결하려 들지 마라 — 그건 확증편향이다.**

## 5. 요약 — 한 장으로

```
Phase 1  경험치 · 레벨업 · 3택1        ← 지금 여기가 다음
   ↓     ★ 5명 테스트. 재플레이 안 하면 STOP
Phase 2  무기 5종 · 적 3종
Phase 3  러닝타임 5~8분 · 보스 · 기록
Phase 4  사운드 · 이펙트 · 화면 흔들림
Phase 5  성능 (느려진 걸 본 다음에)
Phase 6  CrazyGames → Poki
```

### 기억할 한 문장

> **뱀서류의 재미는 총 쏘는 맛이 아니라 "이번엔 어떤 조합이 나올까"에서 나온다.**
> 그러니 레벨업 3택1을 가장 먼저, 가장 공들여 만들어라.
